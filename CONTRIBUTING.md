# Contributing to Project Sentinel

Thank you for your interest in this project. This repository is a personal home lab portfolio designed to demonstrate skills in network security monitoring, firewall engineering, and Infrastructure as Code.

## What This Project Is

- A fully documented, self-contained security operations lab.
- A showcase of practical, hands-on engineering skills.
- A reference architecture for others building similar environments.

## What This Project Is Not

- An active open-source project seeking code contributions.
- A supported or maintained product.
- A production-ready deployment template.

## How You Can Use This Repository

- **Learn and replicate:** Fork the repository and adapt the architecture to your own lab.
- **Reference:** Use the documentation and Terraform modules as inspiration for your own projects.
- **Share feedback:** If you have suggestions, corrections, or questions, feel free to open an issue.

## Pull Requests

While I appreciate the spirit of open-source collaboration, pull requests that modify the core architecture, documentation, or configuration are unlikely to be merged unless they correct a significant error or add substantial educational value.

If you'd like to contribute:

1. Open an issue first to discuss your proposed change.
2. Ensure any code or documentation follows the existing style and tone.
3. Keep changes focused and minimal.

## Code of Conduct

Be respectful, constructive, and professional in all interactions. This is a learning space.

## Questions?

If you have questions about the lab architecture, career advice, or how to build your own, feel free to connect with me on LinkedIn or open a GitHub issue.
